## Design Context

### Users

tmux-mobile is a focused control surface for developers who drive long-running
Codex and Claude Code sessions across several tmux/rmux machines from a phone.
The primary job is to see what an agent is doing, unblock it, and send the next
instruction quickly without turning a narrow mobile screen into a general-purpose
terminal emulator. The default context is one-handed, interruption-heavy use away
from a keyboard; desktop support should remain efficient but follows the mobile
mental model.

### Brand Personality

Calm, direct, and operational. The interface should feel trustworthy and quiet:
the work, terminal output, and agent state are foregrounded while product chrome
recedes. Copy is short and literal. Urgency is reserved for states that genuinely
need the user's attention.

### Aesthetic Direction

Use the existing restrained, Notion-like neutral system in `public/styles.css`:
flat ink-on-paper surfaces, subtle one-pixel borders, small radii, light shadows,
and a single blue accent. Support both the current light and dark themes. Terminal
content uses the established monospace stack; controls use the existing UI type.
Avoid decorative gradients, glass effects, ornamental cards, warm textures, and
new visual motifs that compete with the pane output.

### Design Principles

1. Optimize time-to-unblock: the current agent output and the next useful action
   get the most space and the shortest interaction path.
2. Spend mobile space on content: remove redundant labels and chrome before
   shrinking terminal text or thumb targets.
3. Keep input thumb-reachable and stable: the pane owns the flexible scrolling
   region while the composer remains anchored at the bottom.
4. Preserve context through transient failures: do not blank useful output during
   reconnects, refresh errors, or view changes; show concise state when needed.
5. Keep advanced controls progressively disclosed, while preserving an obvious
   escape path from focused or fullscreen modes.
